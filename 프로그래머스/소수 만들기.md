해당 문제는, 배열에 입력된 수를, 교재 자료에 나와있는 공식을 가지고

소수인지, 아닌지를 검사하고 소수라면 answer을 1씩 늘려서

배열에 소수가 총 몇개 존재하는지를 출력하는 문제이다.

또한 재귀로 구현해야 하나, 많이 복잡한 수를 구하는 것이 아니라고 판단하여

이를 구현하기 위해서 3개의 for 반복문을 사용하여 서로 다른 3 수의 합을 구하고,

또한 해당 반복문 내부에, 만약 그 수의 합이 1이라면 바로 false를 반환하고,

또한 수의 합이 일정한 수로 나누었을 때, 나머지가 0인 즉, 소수가 아니라면 false를

반환하고, if 문이 break되게 된다.

이를 모두 통과했다면 prime 변수의 값이 true 일것이고, answer 변수에 1이

더해지게 되며, 모든 반복문이 종료되었을 때, 소수의 수가 몇인지를 확인할 수 있는

코드를 작성하였다.

첫번째 코드 실행 결과, 프로그래머스 내부 결과값이 12, 20으로 아예 다르게 출력되었다.

이는 반복문이 내가 생각했던 것보다 더 많이 수행되었다고 생각하였고,

for 반복문의 시작 변수의 값을 기존 반복문 변수의 +1 씩으로 수정하였다.

수정한 코드의 실행 결과는 2, 5로 틀린 값을 도출했지만, 기존의 코드보다는

근접한 값을 얻을 수 있었다. 또한 코드를 다시 살펴보았을 때,

나머지가 0인 수를 제외하는 반복문의 조건을 <= 이 아닌 <로 잘못작성하였다는 것을

깨닫고, 이를 수정하여 다시 테스트 해보았다.
