해당 문제는 교재에서 설명했듯이, N 개의 숫자 중에서 서로 다른 3개 즉, 중복 조합이 아닌

경우의 수를 구하는 문제였습니다.

이를 위해 for 반복문 3개를 중첩하여 사용하였습니다.

또한 i,j,k를 반복문의 인수로 사용하여 순서대로 배열의 숫자 3개를 반환하도록 바로 위

반복문의 인수에 +1을 하여 반복하도록 작성하였고

if 조건문으로 해당 3수를 더한 값이 0이라면 경우의 수를 1 더하고

반복문이 종료되도록 작성하였고, 그 후 answer 변수 즉, 경우의 수를

도출하도록 작성하였습니다. 해당 코드실행 결과는 3개의 테스트 중 3개 성공이었습니다.

또한 해당 문제를 중첩반복문이 아니라, 재귀호출로 풀어보았습니다.

중첩반복문으로 풀떄는 간단하게 작성하였지만, 재귀를 해당 코드에 적용하려니

많은 고민이 필요했습니다. 또한 처음에는 최대한 Three 메소드에 파라미터를

적게 할당하여 문제를 풀려고 해보았으나, 자꾸 오류가 발생하고, 파라미터를

적게 넣으려다가 더 많은 전역변수를 선언해야 했기 때문에,

answer 전역변수를 선언하였고

한번의 메소드 호출로

인덱스 위치, 현재 수를 더한 값, 지금까지 몇번 더했는지를 기록하는 변수 3개

그리고 number 배열을 포함해서 4개의 파라미터를 Three 메소드가 받도록 설정하여

수를 3번 더한 합이 0이면 answer의 값을 1 증가시키고, 해당 조건을

만족하지 못한다면, for 반복문 내의 Three 메소드 호출문에서,

각각 변수에 1을 증가시켜 호출하였습니다. 이는 1을 증가시키지 않게되면, 이미 더한 수를

또 한번 더 더하게 되어, 원하는 결과가 도출되지 않고 이는 재귀호출의 목적에 맞지 않는

것이라고 생각하였습니다.

또한 코드의 실행 결과는 성공이었습니다.
