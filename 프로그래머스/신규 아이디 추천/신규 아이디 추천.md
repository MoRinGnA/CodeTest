해당 문제는, 정규식을 여러개 사용하여 풀어야 할 문제로 생각되었습니다.

총 7단계의 id 검사 규칙을 진행해야 하며, 모든 처리과정을 통과한 id를 반환해야 했습니다.

이를 위해서, 기존에 사용한 replaceAll 함수를 적극적으로 사용하였고, 또한 1단계의

모든 대문자를 소문자로 변환하는 것은 toLowerCase 함수를 사용하여 간단하게

해결할 수 있었습니다.

2단계의 경우, 소문자, -, 숫자, \_, . 를 제외한 모든 문자를 제거하는 것이었기에,

[] 내에 ^ 를 사용한, [^-_.a-z0-9] 와 같이 정규식을 구성하였습니다.

또한 3단계의 . 가 연속으로 사용된 부분을 하나의 . 로 대체하는 정규식은

[n,m]을 응용한, [.]{2,} 을 "." 로 대체하도록 replaceAll 함수를 사용하였습니다.

또한 4단계의 처음과 끝 부분에 마침표를 제거하는 경우, ^ 와 $ 를 사용하여

^[.]|[.]$ 로 작성하여, .가 앞에 존재하거나 뒤에 존재한다면 "" 즉, 마침표를

제거하는 정규식을 작성하였습니다.

이후, 5단계인 빈문자열의 경우. if 조건문 length 를 검사하여 빈 문자열의 경우

a를 대입합니다.

6단계는 위 조건문과 비슷하게 길이가 16이 넘어간다면, substring 함수를 사용하여,

처음 15개의 문자를 제외한 나머지 문자를 모두 제거하게 됩니다

7단계는 for 반복문을 사용하여 아이디의 길이가 3이 될때까지 반복하고 종료되게 됩니다.

위 모든 과정을 거친 id는 반환되게 됩니다.

위처럼 작성한 후, 코드 실행 결과는 에러였습니다.

해당 오류는 charAt을 사용하는 과정에서 +=를 사용해야 하는데, =를 사용하여 메소드가

반환한 값을 바로 대입하려고 했기 때문에 발생하였다고 생각하여 이를 수정하고 코드를

실행하니, 5개의 테스트 모두 성공하였습니다.
